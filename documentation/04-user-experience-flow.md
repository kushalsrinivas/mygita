# User Experience Flow

## 🌟 Complete User Journey

This document outlines the detailed user experience flows for My Gita, ensuring every interaction feels intuitive, meaningful, and spiritually enriching.

## 🚪 First-Time User Experience

### Initial App Launch

```
1. Splash Screen (2-3 seconds)
   ├─ Beautiful spiritual imagery (lotus, sunrise, etc.)
   ├─ App logo with gentle fade-in animation
   └─ Loading indicator with spiritual motif

2. Welcome Screen
   ├─ Warm greeting: "Welcome to your spiritual journey"
   ├─ Beautiful background with subtle animations
   ├─ Brief app description (2-3 lines)
   ├─ "Begin Your Journey" button (primary CTA)
   └─ "Learn More" link (optional app tour)
```

### Onboarding Flow

```
1. Profile Setup - Step 1: Personal Information
   ├─ "What should we call you?" (display name input)
   ├─ Character limit indicator (20 chars)
   ├─ Validation with gentle error messages
   └─ "Continue" button (enabled when valid)

2. Profile Setup - Step 2: Avatar Selection
   ├─ Grid of spiritual avatar options
   ├─ Categories: Symbols, Nature, Abstract
   ├─ Preview of selected avatar
   ├─ "Choose Later" option
   └─ "Continue" button

3. Profile Setup - Step 3: Reading Pace
   ├─ "How would you like to pace your journey?"
   ├─ Visual cards for each pace option:
   │   ├─ Casual: "1-2 verses/day" - "Perfect for beginners"
   │   ├─ Balanced: "5 verses/day" - "Steady spiritual growth"
   │   ├─ Intensive: "Full chapter/day" - "Deep immersion"
   │   └─ Custom: "Set your own pace"
   ├─ Estimated completion time for each option
   └─ "Start My Journey" button

4. Permission Requests
   ├─ Notification permission with clear benefits
   ├─ "Stay connected to your daily practice"
   ├─ "Allow" / "Maybe Later" options
   └─ Respect user choice without pressure

5. First Verse Introduction
   ├─ "Here's your first verse" introduction
   ├─ Brief tutorial on reading interface
   ├─ Guided interaction with swipe gestures
   └─ Celebration of first completion
```

## 🔔 Notification Experience

### Notification Types and Timing

```
Daily Reading Reminder:
├─ Trigger: User's chosen daily time
├─ Message: "Your daily verse awaits 🕉️"
├─ Action: Direct to today's verse
└─ Snooze: 30 minutes, 1 hour, 3 hours

Evening Reflection:
├─ Trigger: 2 hours after reading (if no reflection)
├─ Message: "Take a moment to reflect on today's teaching"
├─ Action: Direct to journal
└─ Respect: Don't send if user already reflected

Streak Maintenance:
├─ Trigger: 8 PM if no reading today
├─ Message: "Don't break your [X] day streak! 🔥"
├─ Action: Direct to today's verse
└─ Frequency: Only if streak ≥ 3 days

Achievement Unlock:
├─ Trigger: Immediate upon achievement
├─ Message: "You've unlocked a new badge! 🏆"
├─ Action: Direct to achievement details
└─ Celebration: Include achievement name
```

### Notification Interaction Flow

```
1. Notification Received
   ├─ User sees notification on device
   ├─ Tap notification → Direct app action
   ├─ Swipe away → Gentle reminder later
   └─ Settings → Customize preferences

2. In-App Notification Handling
   ├─ Smooth transition to relevant screen
   ├─ Context preserved from notification
   ├─ No jarring interruptions
   └─ Graceful fallback if content unavailable

3. Notification Customization
   ├─ Time preferences per notification type
   ├─ Weekend/holiday adjustments
   ├─ Do not disturb hours
   └─ Message tone and style options
```
